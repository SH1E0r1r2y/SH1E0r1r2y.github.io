查找檔案 + grep

Level.5

The password for the next level is stored in a file somewhere under the inhere directory and has all of the following properties:
- human-readable
- 1033 bytes in size
- not executable

  1. cd ->ls 後看到一大堆檔案
  2. file ./* 只得到一堆 directory
  3. di -sh ./* 也沒辦法
  4. 經過搜尋，find . - size 1033c 得到位置 ./maybehere07/.file2，cat ./maybehere07/.file2
    • . 代表當前目錄
    • -size [+-]number[cwbkMG]：按文件大小查找，可用 + 、 - 表示大於或小於指定大小，单位可以是 c（bytes）、w（words）、k（KB）、M（MB）或 G（GB）

=>P4L4vucdmLnm8I7Vl7jG1ApGSfjYKqJU

Level.6

The password for the next level is stored somewhere on the server and has all of the following properties:
- owned by user bandit7
- owned by group bandit6
- 33 bytes in size

  1. cd .. 去上一頁，看到超多檔案
  2. 把題目給的條件都打上去find / -type f -user bandit7 -group bandit6 -size 33c，顯示：
    • / ： 在根目錄 /
    • -type f ： 查找所有類型為文件的文件。
    • -user bandit7 ： 查找所有屬於用戶名為 bandit7 的文件。
    • -group bandit6 ： 查找所有所屬組為 bandit6 的文件。
    • -size 33c ： 查找所有大小為 33 個字節的文件。
  3. 使用find / -user bandit7 -group bandit6 -size 33c -type f 2>/dev/null
    • 2>/dev/null - 將錯誤信息重定向到 /dev/null，以便過濾掉不必要的錯誤信息。
  4. 使用find / -type f -user bandit7 -group bandit6 -size 33c 2>&1 | grep -Ev "Permission denied" | grep -Ev "No such file or directory"
    • $()：是一種命令替換語法，用於執行括號內的命令並將其輸出插入到命令行中。
    • 2>&1：是將標準錯誤(stderr)重定向到標準輸出(stdout)
    • grep -Ev "no such file or directory"：查找匹配並過濾掉包含no such file or directory字樣的行，最後顯示符合條件的文件的路徑。

error

  1. find: paths must precede expression: `-‘ 命令格式不正確引起的
  2. bad substitution：哪裡寫錯了

Level.12

The password for the next level is stored in the file data.txt, which is a hexdump of a file that has been repeatedly compressed. For this level it may be useful to create a directory under /tmp in which you can work using mkdir. For example: mkdir /tmp/myname123. Then copy the datafile using cp, and rename it using mv (read the manpages!)

  1. ls+cat

    2.無法直接在原檔案執行 ->在tmp創建一個臨時文件，並複製檔案過來
  2. xxd -r filename：可以把二進位原始檔還原。=> xxd -r data.txt data
  3. 反覆的file、ls、解壓縮
    • 如果file 得出gzip compressed data => data是一個gzip壓縮文件
    • 如果file 得出bzip compressed data => data是一個bzip壓縮文件
    • 如果file 得出POSIX tar archive (GNU) => data 是一個tar 打包文件
  4. 最後得到 ASCII，cat那個檔案

Level.16

The credentials for the next level can be retrieved by submitting the password of the current level to a port on localhost in the range 31000 to 32000. First find out which of these ports have a server listening on them. Then find out which of those speak SSL and which don’t. There is only 1 server that will give the next credentials, the others will simply send back to you whatever you send to it.

  1. cat /etc/bandit_pass/bandit16
  2. nmap -sV -T4 -p 31000-32000 localhost
    • nmap - 網絡探索工具和安全/端口掃描器
    • -T4 提高掃描速度
    • -p 指定端口
    • -sV 識別服務的版本
  3. 試試看31518 & 31790 ，輸入這節的密碼，錯誤的只會回傳相同結果 -> 31790 -> 得到私鑰
  4. mktemp -> cd /tmp/

Level.20

There is a setuid binary in the homedirectory that does the following: it makes a connection to localhost on the port you specify as a commandline argument. It then reads a line of text from the connection and compares it to the password in the previous level (bandit20). If the password is correct, it will transmit the password for the next level (bandit21).

  1. echo -n 'VxCazJaVykI6W36BkBU0mJTCM8rR95XT' | nc -l 5137 &
    • -n ：防止輸入出現換行符
  2. ./suconnect 5137
