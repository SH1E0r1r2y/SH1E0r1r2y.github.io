前置作業

題目轉為 json 格式

  • import re跟import json
  • 打開欲轉換的檔案
  • 使用 正則表達式 找到 match QUESTION NO、Question、Answer:、Explanation:，每組題目是一個 dict，並定義其資料型態，再將 dict append 到 list 裡
    • (.*?) 當要抓取一段不固定的字串時使用
      • ( )：用來定義一個模式以分組
      • .：匹配除換行符外的任何
      • *：匹配前面的模式零次或多次，即可以匹配不固定的的前面字符
      • ?:非貪心量化（Non-greedy quantifiers）：當?緊跟在任何一個其他重複修飾詞（*,+,?，{n}，{n,}，{n,m}）後面時，為 non-greedy match 模式，會匹配盡量少字符
    • (?:)：通常是使用()來創建分組，但如果像這次的 Explanation 一樣部分有部分沒有，就可以使用(?:)，不會強制捕獲匹配的內容，被稱為「非捕獲分組」
    • (?=(?:\nQUESTION NO: \d+)|)：用來確保匹配不會超過下一個 QUESTION NO 的開始，被稱為正向前瞻 (positive lookahead)
    • $：匹配文檔的結尾
    • ()?:表示整個模式是可選的。如果存在 Explanation，就會成功匹配和提取；如果沒有也會印出，只是沒有 Explanation。
  • 讀取現在的output.json，合併並新增資料，獲得 json 腳本
